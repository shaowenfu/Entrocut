# 01. MVP Scope（MVP 范围）

## 1. 产品目标

`EntroCut MVP` 只交付一个闭环：`Chat-to-Cut（对话生成剪辑）`。

用户可以：

1. 导入本地视频素材。
2. 输入自然语言意图（例如“剪成 30 秒高燃混剪”）。
3. 获得可播放预览，并继续通过对话微调。

## 2. 一句话验收标准

用户导入 `1 小时` 素材后，输入目标指令，系统在 `1 分钟` 内给出第一版结构化预览，且支持后续“换个镜头/换音乐/删除镜头”的对话微调。

## 3. MVP In-Scope（范围内）

1. `Ingest Pipeline（素材摄入管线）`
   1. 本地镜头切分（`PySceneDetect`）。
   2. 每镜头 4 帧关键帧抽样与拼图（含位置编号）。
   3. 通过云端完成向量化与索引入库。
2. `Agent Orchestration（智能体编排）`
   1. 理解用户意图。
   2. 生成检索请求与剪辑计划。
   3. 生成 `EntroVideoProject` 契约。
3. `Render Preview（预览渲染）`
   1. 基于契约生成可播放预览流。
   2. 支持对话微调后快速重渲染。
4. `Client Workspace（客户端工作台）`
   1. 启动页（项目管理）。
   2. 工作台：左侧 `AI Copilot（AI 助手）`，右侧素材与预览区。
5. `Export（导出）`
   1. 仅支持单一默认配置。
   2. 导出期间禁止并行编辑。

## 4. MVP Out-of-Scope（范围外）

1. 复杂时间线操作（高级拖拽、关键帧动画、复杂特效）。
2. 多用户协作编辑（仅预留 `Multi-User Interface（多用户接口）`）。
3. 云端素材存储（素材全本地）。
4. 通用 `Tool/Skill（工具/技能）` 执行框架的完整实现。
5. `Undo/Redo（撤销/重做）`、快捷键体系、插件系统。

## 5. 关键原则

1. `AI-first（AI 优先）`：编辑器先服务于 AI 决策可视化，不追求完整 NLE 功能。
2. `Local-first（本地优先）`：原始视频不上传云端。
3. `Schema-first（结构先行）`：先定契约，再写实现。
4. `No Hidden Decisions（无黑盒决策）`：关键镜头选择必须给出 `reasoning（选择理由）`。
