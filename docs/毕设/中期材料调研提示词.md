# 中期材料调研提示词

这份文档用于指导 `Manus` 等智能体完成中期材料相关的文献调研、外文翻译选文与资料整理。

当前课题在学校口径上仍以“**主题驱动的多源视频自动编排系统设计与实现**”为主线，但结合仓库当前实际进展，调研范围不能只覆盖传统的视频检索、摘要与自动剪辑，还必须纳入 `AI agent design and development（AI 智能体设计与开发）`、运行时架构、工具调用闭环、上下文组装与工程实现等内容。

目标不是做一篇“全而大”的学术综述，而是为今晚的中期材料提交服务：快速整理出一套**能支撑文献综述、中期检查表、外文翻译选文和后续毕业设计说明文档**的资料包。

## 一、调研目标

需要支撑的提交物包括：

1. `文献综述`
2. `外文文献翻译`
3. `中期检查表 / 中期报告`

调研输出至少要回答下面几个问题：

1. 主题驱动视频自动编排的研究问题是什么。
2. 这一问题通常拆成哪些技术子问题。
3. 当前主流方法有哪些，分别解决什么问题。
4. `AI agent` 在视频编辑或复杂多步骤媒体任务中起什么作用。
5. 当前项目的技术路线与已有研究之间的关系是什么。
6. 哪些英文材料最适合作为本科毕业设计的外文翻译材料。

## 二、调研范围

文献调研按 6 个主题块组织：

### 1. `Text-to-Video Retrieval（文本到视频检索）`

关注自然语言主题如何映射到视频片段检索。

### 2. `Video Summarization（视频摘要）`

关注长视频压缩、关键信息提取、用户导向摘要与候选片段筛选。

### 3. `Shot Boundary Detection / Segmentation（镜头边界检测 / 视频切分）`

关注如何把原始视频切成适合后续检索和编排的语义片段。

### 4. `Automatic Video Editing（自动视频剪辑）`

关注自动选材、排序、拼接、主题一致性与最终导出。

### 5. `AI Agent Design and Development（AI 智能体设计与开发）`

这是本次补充的重点。关注：

1. `agent runtime（智能体运行时）`
2. `planner-executor architecture（规划器-执行器架构）`
3. `tool calling（工具调用）`
4. `memory / context assembly（记忆 / 上下文组装）`
5. `human-in-the-loop（人在回路）`
6. `state management（状态管理）`
7. 多步骤任务闭环

### 6. `System / Pipeline Engineering（系统与工程管线）`

关注 `client-core-server`、接口契约、服务端能力、测试与工程闭环。

## 三、检索策略

### 1. 检索顺序

先按下面顺序搜，避免一开始陷入细节：

1. 先找 2-3 篇 `survey / review（综述）`
2. 再找 5-8 篇 `foundational papers（基础论文）`
3. 再补 3-5 篇 2023-2025 的新工作
4. 最后补 2-4 个工程实现或系统设计资料

### 2. 优先来源

1. `Google Scholar`
2. `Semantic Scholar`
3. `arXiv`
4. `IEEE Xplore`
5. `ACM Digital Library`
6. `CVPR / ICCV / ECCV / AAAI / ACM MM / NeurIPS / ICLR`

### 3. 筛选标准

优先保留：

1. 与“文本驱动的视频检索、筛选、编排”直接相关的工作
2. 与 `AI agent` 架构、工具调用、多步执行闭环相关的工作
3. 既能写进文献综述，又能支撑工程实现叙事的资料
4. 篇幅适中、语言相对平实、适合作为外文翻译材料的论文

尽量排除：

1. 纯 `Text-to-Video Generation（文生视频生成）`
2. 偏艺术理论、不支撑系统实现的影视研究
3. 与当前课题关系很弱的大而泛 `LLM` 论文

## 五、给 Manus 的完整调研提示词

下面是面向文献调研的主提示词。

```text
请围绕“主题驱动的多源视频自动编排系统设计与实现”做一份面向本科毕业设计中期材料的文献调研。

注意：这个课题虽然名义上是视频自动编排系统，但项目当前实际推进中还包含 AI agent 运行时、工具调用闭环、上下文组装、服务端能力设计等内容，因此调研范围不能只覆盖传统视频方向，还要覆盖 AI agent design and development。

请按以下 6 个方向组织调研：
1. Text-to-Video Retrieval（文本到视频检索）
2. Video Summarization（视频摘要）
3. Shot Boundary Detection / Segmentation（镜头边界检测 / 视频切分）
4. Automatic Video Editing（自动视频剪辑）
5. AI Agent Design and Development（AI 智能体设计与开发）
6. System / Pipeline Engineering（系统与工程管线）

调研要求：
1. 优先收集 2021-2025 的英文论文，同时保留少量经典基础论文。
2. 优先找 survey / review，其次找 foundational papers，再找近两年的代表性工作。
3. 论文来源优先 Google Scholar、Semantic Scholar、arXiv、IEEE、ACM、CVPR/ICCV/ECCV/AAAI/ACM MM。
4. 避免把重点放在纯 Text-to-Video Generation，而应聚焦“检索、筛选、编排、系统闭环、agent runtime”。

请输出一张结构化表格，字段包括：
- title
- year
- venue
- category
- problem
- method
- contribution
- limitations
- relation to my project
- whether suitable for literature review citation
- whether suitable as foreign translation material

请保证：
1. 每个方向至少给出 3 篇候选论文
2. 总量控制在 15-20 篇
3. 单独标出：
   - 2 篇最适合写文献综述的 survey / review
   - 3 篇最关键的基础论文
   - 2 篇最适合做外文翻译的材料

在输出表格之后，请继续输出：
1. 一份适合本科毕业设计中期材料的文献综述提纲
2. 各主题块之间的关系图或文字化结构
3. 哪些内容最适合写进“实验进展”和“其它情况”
4. 如何把 AI agent design and development 合理纳入本课题叙事，而不显得偏题
```

## 六、给 Manus 的“文献综述提炼”提示词

当 `Manus` 已经找回一批论文后，再使用下面这个提示词做二次加工。

```text
我已经收集了一批与“主题驱动的多源视频自动编排系统设计与实现”相关的论文。请你基于这些论文，帮我整理一版适合本科毕业设计中期材料提交的文献综述。

要求：
1. 结构上按照下面几个主题块展开：
   - Text-to-Video Retrieval
   - Video Summarization
   - Shot Boundary Detection / Segmentation
   - Automatic Video Editing
   - AI Agent Design and Development
   - System / Pipeline Engineering
2. 强调这些方向如何共同支撑“主题驱动的视频自动编排系统”。
3. 明确说明 AI agent 相关内容不是偏题，而是系统从传统自动剪辑走向智能化工作流的重要扩展。
4. 语言风格适合本科毕业设计，不要写成过度空泛的 AI 宣传稿。
5. 输出长度控制在 1200-2000 字。

请输出：
1. 文献综述正文
2. 可引用的参考文献列表
3. 每篇参考文献在综述中的作用说明
```

2. 最好能覆盖 AI agent 或系统设计视角中的一个方面
3. 篇幅适中，不要过长
4. 语言相对平实，适合翻译
5. 适合作为学校中期检查提交材料

请输出：
- 论文标题
- 链接
- 作者
- 年份
- 推荐理由

## 八、给 Manus 的“外文翻译执行”提示词

当你已经选定一篇论文后，再用下面这个提示词。

```text
我已经选定一篇与“主题驱动的多源视频自动编排系统设计与实现”相关的英文文献，请你帮我完成外文翻译整理。

要求：
1. 先输出论文的基本信息：
   - title
   - authors
   - year
   - venue
   - why it matches my project
2. 再给出这篇论文最适合用于本科毕业设计外文翻译提交的章节建议。
3. 如果篇幅太长，请推荐一个合理的翻译范围，并说明为什么。
4. 输出中文译文时，语言尽量准确、通顺、术语统一。
5. 专业术语保留英文，并在首次出现时补中文说明。
6. 最后再给出一个“这篇论文对我当前项目有什么启发”的总结段落。

请注意：
1. 我的项目除了视频自动编排，还涉及 AI agent runtime、tool calling、context assembly、server capability 等工程内容。
2. 因此请在总结中指出，该文献与视频系统、智能体架构或工程闭环分别有什么关系。
```

## 九、给 Manus 的“中期检查表支撑材料提炼”提示词

这段提示词用于把调研结果转成中期表格和系统提交材料能用的语言。

```text
请基于当前课题“主题驱动的多源视频自动编排系统设计与实现”的文献调研结果，帮我提炼出适合写入本科毕业设计中期检查表和中期提交系统的表达。

请分别生成：
1. 文献综述完成情况说明
2. 实验进展情况说明
3. 其它情况说明
4. 课题完成情况说明

写作要求：
1. 语气要真实稳妥，不要夸大
2. 既体现当前项目已经完成的设计、文档、架构和部分开发工作
3. 又承认系统仍处于集成和完善阶段
4. 特别把 AI agent 设计与开发解释为系统智能化能力建设的一部分，而不是偏离课题
5. 每段控制在 120-220 字左右
```

## 十、期望 Manus 返回的成果格式

无论是文献调研还是翻译选文，都尽量要求 `Manus` 输出为下面这种格式：

### 1. 论文清单表

至少包含：

- `title`
- `year`
- `venue`
- `category`
- `problem`
- `method`
- `relation to project`
- `translation suitability`

### 2. 综述提纲

至少包含：

1. 研究背景
2. 关键技术方向
3. `AI agent` 相关扩展
4. 当前系统实现与已有研究的关系
5. 小结

### 3. 选文结论

至少包含：

1. 最推荐的 1 篇外文翻译材料
2. 备选的 1-2 篇材料
3. 选择理由
4. 风险点

## 十一、使用建议

建议你今天按这个顺序跑：

1. 先跑“完整调研提示词”
2. 再跑“外文翻译选文提示词”
3. 然后把 `Manus` 返回的候选论文清单贴回来
4. 最后再用“文献综述提炼提示词”和“中期检查表支撑材料提炼提示词”做收口

如果时间特别紧，优先级如下：

1. 先搞定外文翻译选文
2. 再补文献综述提纲
3. 最后再精修中期检查表文字
