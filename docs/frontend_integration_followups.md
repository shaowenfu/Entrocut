# Frontend Integration Follow-ups

日期：2026-03-07

背景：

当前 `Launchpad -> Workspace` 已经接到真实 `Core API/WS contract`，并完成了最小联调。创建项目、进入工作台、读取 `workspace snapshot` 已经可用。

本轮联调新发现的问题：

## 1. 素材与片段缺少封面图

现象：

1. 上传视频后，`Assets` 区域没有显示素材封面
2. `AI Storyboard` 下方的片段卡片也没有显示封面

影响：

1. 用户无法快速辨认素材与片段内容
2. 工作台当前只具备结构展示，缺少视觉可用性

当前判断：

1. 这是前端展示能力缺失，不是 `Core contract` 阻塞项
2. 后续需要明确封面来源：
   - 临时占位图 / 渐变封面
   - 本地视频首帧截图
   - `Core` 返回缩略图字段

## 2. 点击素材/片段/分镜卡片后，预览播放器没有联动播放

现象：

1. 点击 `Assets` 或 `Clips` 项目，不会在播放器窗口播放对应素材/片段
2. 点击 `AI Storyboard` 下方镜头卡片，也不会跳转到对应片段播放

期望行为：

1. 点击素材时，播放器切换到该素材并开始播放
2. 点击片段时，播放器跳转到对应素材的起止区间并开始播放
3. 点击 `Storyboard` 镜头卡片时，播放器同步跳到关联片段

影响：

1. 当前工作台缺少核心“看见结果”的交互闭环
2. `Storyboard -> Preview` 之间还没有形成真正的编辑反馈链路

当前判断：

1. 这是工作台预览层能力缺失
2. 后续需要补一套最小 `preview state` 与卡片联动规则：
   - 当前播放素材 `activeAssetId`
   - 当前播放片段 `activeClipId`
   - 当前预览时间 `currentTime`
   - `StoryboardScene -> Clip` 的最小映射关系

非目标：

1. 本轮不做真实视频解码、时间轴编辑器、精确帧级控制
2. 本轮只记录问题，不在当前提交中修复
